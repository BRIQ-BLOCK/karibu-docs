Create, manage, and track organized SMS campaigns within your workspaces. Campaigns help you coordinate targeted messaging initiatives with scheduling, audience segmentation, and detailed analytics.

## Overview

Campaigns are structured SMS initiatives that allow you to organize and track your messaging efforts. Each campaign belongs to a workspace and can contain multiple message batches, scheduled sends, and detailed performance metrics.

### Campaign Benefits

- **Organization**: Group related messages under one campaign
- **Scheduling**: Plan and schedule messages for optimal delivery
- **Targeting**: Send to specific groups and segments
- **Analytics**: Track performance and ROI metrics
- **Collaboration**: Team members can collaborate on campaign execution

### Base URL
```
https://karibu.briq.tz/
```

### Authentication
```http
X-API-Key: <your_api_key>
Content-Type: application/json
```

## Create Campaign

Create a new campaign within a specific workspace.

### Endpoint
```http
POST /v1/campaign/create/
```

### Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `workspace_id` | string | Yes | UUID of the workspace to create campaign in |
| `name` | string | Yes | Campaign name (3-100 characters) |
| `description` | string | No | Campaign description (max 500 characters) |
| `launch_date` | string | No | ISO 8601 timestamp for campaign launch |
| `end_date` | string | No | ISO 8601 timestamp for campaign end |
| `budget_limit` | integer | No | Budget limit in TZS |
| `tags` | array | No | Array of tags for campaign organization |

### Example Request

```bash
curl -X POST https://karibu.briq.tz/v1/campaign/create/ \
-H "Content-Type: application/json" \
-H "X-API-Key: your_api_key_here" \
-d '{
  "workspace_id": "ws_abc123xyz789def",
  "name": "Summer Sale 2024",
  "description": "Promotional campaign for summer product discounts",
  "launch_date": "2024-12-01T08:00:00Z",
  "end_date": "2024-12-31T23:59:59Z",
  "budget_limit": 1000000,
  "tags": ["summer", "sale", "promotion"]
}'
```

### Response

## Get All Campaigns

```http
GET /v1/campaign/all/
```

| `workspace_id` | string | No | Filter by specific workspace |
| `status` | string | No | Filter by status: draft, active, paused, completed, cancelled |
| `limit` | integer | No | Number of campaigns to retrieve (default: 50, max: 1000) |
| `offset` | integer | No | Number of campaigns to skip (for pagination) |
| `tags` | string | No | Comma-separated tags to filter by |
| `date_from` | string | No | Filter campaigns created from date (YYYY-MM-DD) |

## Campaign Status Management

### Status Transitions

| From Status | To Status | Description |
|-------------|-----------|-------------|
| `draft` | `active` | Launch the campaign |
| `active` | `paused` | Temporarily pause campaign |
| `paused` | `active` | Resume paused campaign |
| `active` | `completed` | Mark campaign as completed |
| `draft` | `cancelled` | Cancel draft campaign |
| `active` | `cancelled` | Cancel active campaign |

### Status Update Example

```bash
curl -X PATCH https://karibu.briq.tz/v1/campaign/update/camp_def456ghi789jkl \
-H "Content-Type: application/json" \
-H "X-API-Key: your_api_key_here" \
-d '{
  "status": "paused"
}'
```

## Campaign Analytics

### Performance Metrics

Access detailed campaign performance through the campaign details endpoint:

- **Delivery Metrics**: Sent, delivered, failed message counts
- **Engagement Metrics**: Click-through rates, conversion rates
- **Financial Metrics**: Cost per message, ROI, revenue attribution
- **Audience Metrics**: Reach, frequency, unique recipients
- **Time-based Analysis**: Performance trends over campaign duration

### Custom Analytics Query

```bash
curl -X GET "https://karibu.briq.tz/v1/campaign/camp_def456ghi789jkl/analytics?metric=delivery_rate&period=daily" \
-H "X-API-Key: your_api_key_here"
```

```python
import requests
import json
from datetime import datetime, timedelta

class BriqCampaign:
    def __init__(self, api_key):
        self.api_key = api_key
        self.base_url = 'https://karibu.briq.tz'
        self.headers = {
            'X-API-Key': api_key,
            'Content-Type': 'application/json'
        }
    
    def create_campaign(self, workspace_id, campaign_data):
        url = f'{self.base_url}/v1/campaign/create/'
        data = {
            'workspace_id': workspace_id,
            **campaign_data
        }
        
        try:
            response = requests.post(url, headers=self.headers, data=json.dumps(data))
            response.raise_for_status()
            return response.json()
        except requests.exceptions.RequestException as e:
            print(f'Error creating campaign: {e}')
            return None
    
    def get_all_campaigns(self, **filters):
        url = f'{self.base_url}/v1/campaign/all/'
        
        try:
            response = requests.get(url, headers={'X-API-Key': self.api_key}, params=filters)
            response.raise_for_status()
            return response.json()
        except requests.exceptions.RequestException as e:
            print(f'Error fetching campaigns: {e}')
            return None
    
    def get_campaign(self, campaign_id):
        url = f'{self.base_url}/v1/campaign/{campaign_id}/'
        
        try:
            response = requests.get(url, headers={'X-API-Key': self.api_key})
            response.raise_for_status()
            return response.json()
        except requests.exceptions.RequestException as e:
            print(f'Error fetching campaign: {e}')
            return None
    
    def update_campaign(self, campaign_id, **updates):
        url = f'{self.base_url}/v1/campaign/update/{campaign_id}'
        
        try:
            response = requests.patch(url, headers=self.headers, data=json.dumps(updates))
            response.raise_for_status()
            return response.json()
        except requests.exceptions.RequestException as e:
            print(f'Error updating campaign: {e}')
            return None
    
    def launch_campaign(self, campaign_id):
        return self.update_campaign(campaign_id, status='active')
    
    def pause_campaign(self, campaign_id):
        return self.update_campaign(campaign_id, status='paused')
    
    def complete_campaign(self, campaign_id):
        return self.update_campaign(campaign_id, status='completed')
    
    def get_campaign_performance(self, campaign_id):
        campaign = self.get_campaign(campaign_id)
        if campaign and campaign.get('success'):
            stats = campaign.get('statistics', {})
            metrics = campaign.get('performance_metrics', {})
            return {
                'delivery_rate': stats.get('delivery_rate', 0),
                'roi_percentage': metrics.get('roi_percentage', 0),
                'cost_efficiency': metrics.get('cost_per_delivery', 0),
                'total_reach': stats.get('unique_recipients', 0)
            }
        return None
```

```php
<?php
class BriqCampaign {
    private $apiKey;
    private $baseURL = 'https://karibu.briq.tz';
    
    public function __construct($apiKey) {
        $this->apiKey = $apiKey;
    }
    
    private function makeRequest($method, $endpoint, $data = null) {
        $url = $this->baseURL . $endpoint;
        $headers = [
            'X-API-Key: ' . $this->apiKey,
            'Content-Type: application/json'
        ];
        
        $ch = curl_init();
        curl_setopt($ch, CURLOPT_URL, $url);
        curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
        curl_setopt($ch, CURLOPT_HTTPHEADER, $headers);
        curl_setopt($ch, CURLOPT_CUSTOMREQUEST, $method);
        
        if ($data) {
            curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
        }
        
        $response = curl_exec($ch);
        $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
        curl_close($ch);
        
        return [
            'status' => $httpCode,
            'data' => json_decode($response, true)
        ];
    }
    
    public function createCampaign($workspaceId, $campaignData) {
        $data = array_merge(['workspace_id' => $workspaceId], $campaignData);
        return $this->makeRequest('POST', '/v1/campaign/create/', $data);
    }
    
    public function getAllCampaigns($filters = []) {
        $queryString = http_build_query($filters);
        $endpoint = '/v1/campaign/all/' . ($queryString ? '?' . $queryString : '');
        return $this->makeRequest('GET', $endpoint);
    }
    
    public function getCampaign($campaignId) {
        return $this->makeRequest('GET', '/v1/campaign/' . $campaignId . '/');
    }
    
    public function updateCampaign($campaignId, $updates) {
        return $this->makeRequest('PATCH', '/v1/campaign/update/' . $campaignId, $updates);
    }
    
    public function launchCampaign($campaignId) {
        return $this->updateCampaign($campaignId, ['status' => 'active']);
    }
    
    public function pauseCampaign($campaignId) {
        return $this->updateCampaign($campaignId, ['status' => 'paused']);
    }
    
    public function completeCampaign($campaignId) {
        return $this->updateCampaign($campaignId, ['status' => 'completed']);
    }
}
```

```json
    "message": "Campaign budget limit has been exceeded",
    "details": {
```

### Error Codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `INVALID_API_KEY` | 401 | API key is missing or invalid |
| `CAMPAIGN_NOT_FOUND` | 404 | Campaign with given ID doesn't exist |
| `WORKSPACE_NOT_FOUND` | 404 | Specified workspace doesn't exist |

| `INVALID_DATE_RANGE` | 400 | Launch/end dates are invalid |
| `CAMPAIGN_ALREADY_ACTIVE` | 409 | Cannot modify active campaign |
| `PERMISSION_DENIED` | 403 | Insufficient permissions for workspace |
| `NAME_ALREADY_EXISTS` | 409 | Campaign name already exists in workspace |

## Best Practices

### Campaign Planning
- Set realistic budgets with 10-20% buffer
- Plan launch dates considering time zones
- Use descriptive names and detailed descriptions
- Tag campaigns consistently for better organization

### Content Strategy
- Test messages before full campaign launch
- Keep content relevant to campaign goals
- Include clear call-to-action in messages
- Consider message length and readability

### Audience Management
- Segment audiences for targeted messaging
- Respect opt-out preferences
- Clean recipient lists regularly
- Monitor delivery rates and adjust accordingly

### Performance Optimization
- Monitor campaigns in real-time
- Use A/B testing for optimization
- Track ROI and conversion metrics
- Adjust timing based on engagement data

## Campaign Limits

### Account Limits
- **Free Account**: 5 campaigns per workspace
- **Starter Plan**: 25 campaigns per workspace  
- **Business Plan**: 100 campaigns per workspace
- **Enterprise Plan**: Unlimited campaigns

### Campaign Limits
- Maximum 100,000 recipients per campaign
- Maximum 50 message batches per campaign
- Campaign name must be unique within workspace
- Maximum 1 year duration per campaign
