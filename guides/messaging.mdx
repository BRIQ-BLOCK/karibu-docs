# Messaging

Briq provides two main approaches for sending SMS messages: instant messages for immediate delivery and campaigns for organized bulk messaging within workspaces.

## Message Types

### Instant Messages
- **Purpose**: Immediate, one-time messages
- **Use cases**: Notifications, alerts, transactional messages
- **Recipients**: Individual phone numbers
- **Delivery**: Immediate sending

### Campaign Messages  
- **Purpose**: Organized messaging within workspaces
- **Use cases**: Marketing campaigns, bulk communications
- **Recipients**: Phone number lists
- **Management**: Created and managed within workspaces

## Sending Instant Messages

### Basic Instant Message

```bash
curl -X POST https://karibu.briq.tz/v1/messages/instant/ \
-H "Content-Type: application/json" \
-H "X-API-Key: your_api_key_here" \
-d '{
  "phone_number": "255712345678",
  "message": "Hello! This is a test message from Briq.",
  "sender_id": "YourBrand"
}'
```

**Response:**
```json
{
  "success": true,
  "message_id": "msg_abc123xyz789def",
  "phone_number": "255712345678",
  "message": "Hello! This is a test message from Briq.",
  "sender_id": "YourBrand",
  "status": "sent",
  "created_at": "2024-08-10T10:30:00Z"
}
```

### Batch Instant Messages

Send to multiple recipients at once:

```bash
curl -X POST https://karibu.briq.tz/v1/messages/send-batch \
-H "Content-Type: application/json" \
-H "X-API-Key: your_api_key_here" \
-d '{
  "recipients": ["255712345678", "255698765432", "255611223344"],
  "message": "System maintenance scheduled for tonight 11 PM - 2 AM. Expect service interruption.",
  "sender_id": "SUPPORT"
}'
```

### Instant Message Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `phone_number` | string | Yes | Recipient's phone number (with country code) |
| `message` | string | Yes | SMS message content (max 160 characters for single SMS) |
| `sender_id` | string | Yes | Your brand name or sender identifier |

## Managing Campaigns

Campaigns are created within workspaces and allow you to organize and send bulk messages.

### Creating a Campaign

```bash
curl -X POST https://karibu.briq.tz/v1/campaigns/create/ \
-H "Content-Type: application/json" \
-H "X-API-Key: your_api_key_here" \
-d '{
  "workspace_id": "ws_abc123xyz789def",
  "name": "Summer Sale Campaign",
  "description": "Promotional campaign for summer discounts"
}'
```

**Response:**
```json
{
  "success": true,
  "campaign_id": "camp_xyz789abc123def",
  "workspace_id": "ws_abc123xyz789def", 
  "name": "Summer Sale Campaign",
  "description": "Promotional campaign for summer discounts",
  "created_at": "2024-08-10T10:30:00Z"
}
```

### Listing Campaigns

```bash
curl -X GET https://karibu.briq.tz/v1/campaigns/all/ \
-H "X-API-Key: your_api_key_here"
```

### Getting Campaign Details

```bash
curl -X GET https://karibu.briq.tz/v1/campaigns/camp_xyz789abc123def \
-H "X-API-Key: your_api_key_here"
```

### Updating a Campaign

```bash
curl -X PATCH https://karibu.briq.tz/v1/campaigns/update/camp_xyz789abc123def \
-H "Content-Type: application/json" \
-H "X-API-Key: your_api_key_here" \
-d '{
  "name": "Updated Summer Sale Campaign",
  "description": "Updated promotional campaign description"
}'
```

## Best Practices

### Message Content
1. **Keep it concise**: SMS has character limits, be clear and direct
2. **Include sender identification**: Always use a recognizable sender ID
3. **Add value**: Make sure your message provides useful information
4. **Call to action**: Include clear next steps if needed

### Phone Number Format
- Always include country code (e.g., 255 for Tanzania)
- Use format: 255712345678 (no spaces or special characters)
- Validate numbers before sending to avoid errors

### Campaign Organization
1. **Use descriptive names**: Make campaigns easy to identify
2. **Workspace separation**: Keep different projects in separate workspaces
3. **Regular cleanup**: Archive or delete old campaigns periodically

### Error Handling
- Always check response status before proceeding
- Implement retry logic for failed messages
- Log errors for debugging and monitoring
- Validate phone numbers before sending
