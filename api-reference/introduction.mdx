Setting up BRIQ SMS API integration is straightforward and can be accomplished in multiple programming languages. BRIQ provides a RESTful API that works with any HTTP client, making it easy to integrate SMS functionality into your existing applications.

## API Requirements

Before getting started, ensure you have:
- A BRIQ account with API access
- Your API key from the BRIQ dashboard
- Internet connectivity for API calls
- A development environment with HTTP client capabilities

## Authentication Setup

All BRIQ API requests require authentication using your API key. You'll need to include this key in the header of every request.

<Stepper>
  <StepperItem title="Step 1: Obtain Your API Key">
    Log into your BRIQ dashboard and navigate to the API settings to generate your API key:
    ```plaintext
    Dashboard → Settings → API Keys → Generate New Key
    ```
  </StepperItem>
  <StepperItem title="Step 2: Store Your API Key Securely">
    Save your API key in environment variables for security:
    ```bash
    export BRIQ_API_KEY="your_api_key_here"
    ```
    <Note title="Security">
      Never hardcode API keys in your source code. Always use environment variables or secure configuration files.
    </Note>
  </StepperItem>
  <StepperItem title="Step 3: Set Base URL">
    Configure the BRIQ API base URL in your application:
    ```plaintext
    Base URL: https://karibu.briq.tz/
    ```
  </StepperItem>
  <StepperItem title="Step 4: Test Authentication">
    Test your setup with a simple API call to verify authentication:
    ```bash
    curl -X GET "https://karibu.briq.tz/v1/workspace/all/" \
    -H "X-API-Key: your_api_key_here"
    ```
  </StepperItem>
</Stepper>

For testing and quick integration, you can use cURL to interact with the BRIQ API:

```bash

```

## JavaScript/Node.js Integration

<Stepper>
  <StepperItem title="Step 1: Install HTTP Client">
    Install axios for making HTTP requests:
    ```bash
    npm install axios
    ```
  </StepperItem>
  <StepperItem title="Step 2: Create BRIQ Client">
    Set up a basic BRIQ API client:
    ```javascript
    const axios = require('axios');

    class BriqClient {
      constructor(apiKey) {
        this.apiKey = apiKey;
        this.baseURL = 'https://karibu.briq.tz';
        this.headers = {
          'X-API-Key': apiKey,
          'Content-Type': 'application/json'
        };
      }

      async sendSMS(content, recipients, senderId = 'BRIQ') {
        try {
          const response = await axios.post(`${this.baseURL}/v1/message/send-instant`, {
            content,
            recipients,
            sender_id: senderId
          }, { headers: this.headers });
          
          return response.data;
        } catch (error) {
          throw new Error(`SMS sending failed: ${error.response?.data?.message || error.message}`);
        }
      }
    }

    module.exports = BriqClient;
    ```
  </StepperItem>
  <StepperItem title="Step 3: Test Your Integration">
    Test your setup with a simple message:
    ```javascript
    const BriqClient = require('./briq-client');

    const briq = new BriqClient(process.env.BRIQ_API_KEY);

    briq.sendSMS('Hello from Node.js!', ['255788344348'])
      .then(result => console.log('Success:', result))
      .catch(error => console.error('Error:', error.message));
    ```
  </StepperItem>
</Stepper>

## Python Integration

<Stepper>
  <StepperItem title="Step 1: Install Requests Library">
    Install the requests library for HTTP operations:
    ```bash
    pip install requests
    ```
  </StepperItem>
  <StepperItem title="Step 2: Create BRIQ Client Class">
    Set up a Python client for BRIQ API:
    ```python
    import requests
    import json
    import os

    class BriqClient:
        def __init__(self, api_key=None):
            self.api_key = api_key or os.getenv('BRIQ_API_KEY')
            self.base_url = 'https://karibu.briq.tz'
            self.headers = {
                'X-API-Key': self.api_key,
                'Content-Type': 'application/json'
            }
        
        def send_sms(self, content, recipients, sender_id='BRIQ'):
            url = f'{self.base_url}/v1/message/send-instant'
            data = {
                'content': content,
                'recipients': recipients,
                'sender_id': sender_id
            }
            
            try:
                response = requests.post(url, headers=self.headers, data=json.dumps(data))
                response.raise_for_status()
                return response.json()
            except requests.exceptions.RequestException as e:
                raise Exception(f'SMS sending failed: {e}')
    ```
  </StepperItem>
  <StepperItem title="Step 3: Test Your Setup">
    Verify your Python integration:
    ```python
    # Test the client
    briq = BriqClient()

    try:
        result = briq.send_sms('Hello from Python!', ['255788344348'])
        print('SMS sent successfully:', result)
    except Exception as e:
        print('Error:', str(e))
    ```
  </StepperItem>
</Stepper>

## PHP Integration

<Stepper>
  <StepperItem title="Step 1: Verify cURL Extension">
    Ensure PHP cURL extension is installed:
    ```bash
    php -m | grep curl
    ```
  </StepperItem>
  <StepperItem title="Step 2: Create BRIQ Client Class">
    Set up a PHP client for BRIQ API:
    ```php
    <?php
    class BriqClient {
        private $apiKey;
        private $baseURL = 'https://karibu.briq.tz';
        
        public function __construct($apiKey = null) {
            $this->apiKey = $apiKey ?: getenv('BRIQ_API_KEY');
        }
        
        public function sendSMS($content, $recipients, $senderId = 'BRIQ') {
            $url = $this->baseURL . '/v1/message/send-instant';
            $data = [
                'content' => $content,
                'recipients' => $recipients,
                'sender_id' => $senderId
            ];
            
            $ch = curl_init();
            curl_setopt($ch, CURLOPT_URL, $url);
            curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
            curl_setopt($ch, CURLOPT_HTTPHEADER, [
                'X-API-Key: ' . $this->apiKey,
                'Content-Type: application/json'
            ]);
            curl_setopt($ch, CURLOPT_POST, true);
            curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
            
            $response = curl_exec($ch);
            $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
            curl_close($ch);
            
            if ($httpCode !== 200) {
                throw new Exception('SMS sending failed: ' . $response);
            }
            
            return json_decode($response, true);
        }
    }
    ?>
    ```
  </StepperItem>
  <StepperItem title="Step 3: Test PHP Integration">
    Test your PHP setup:
    ```php
    <?php
    require_once 'briq-client.php';

    try {
        $briq = new BriqClient();
        $result = $briq->sendSMS('Hello from PHP!', ['255788344348']);
        echo 'SMS sent successfully: ' . json_encode($result);
    } catch (Exception $e) {
        echo 'Error: ' . $e->getMessage();
    }
    ?>
    ```
  </StepperItem>
</Stepper>

## Environment Configuration

### Development Environment

Set up your development environment variables:

```bash
# .env file
BRIQ_API_KEY=your_development_api_key
BRIQ_BASE_URL=https://karibu.briq.tz
BRIQ_DEFAULT_SENDER=BRIQ
BRIQ_TIMEZONE=Africa/Dar_es_Salaam
```

### Production Environment

For production deployments:

```bash
# Production environment variables
BRIQ_API_KEY=your_production_api_key
BRIQ_BASE_URL=https://karibu.briq.tz
BRIQ_DEFAULT_SENDER=YOURCOMPANY
BRIQ_WEBHOOK_URL=https://yourapp.com/webhooks/briq
```

<Note title="Environment Separation">
Always use separate API keys for development and production environments to maintain security and prevent accidental charges during testing.
</Note>

## Common Installation Issues

### Invalid API Key Error
If you receive authentication errors:
1. Verify your API key is correct
2. Check that the key is properly formatted in headers
3. Ensure you're using the correct environment (dev/prod)

### Network Connectivity Issues
For connection problems:
1. Verify internet connectivity
2. Check firewall settings
3. Ensure HTTPS requests are allowed
4. Test with cURL first

### Rate Limiting
If you encounter rate limits:
1. Implement exponential backoff
2. Add delays between requests
3. Use batch endpoints for multiple operations
4. Contact support for limit increases
